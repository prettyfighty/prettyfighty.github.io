    Ruby裡 String 和 Symbol 的異同

    小知識 ruby
    Created At : 2021-05-15 17:01
    1. 相同點
      1. capitalize, upcase, downcase
      2. empty?, start_with?, end_with?
      3. length, size, []
    2. 不同點
      1. String可改變；Symbol不可變
      2. object_id
    3. 相互轉換
      1. to_s 轉成字串
      2. to_sym 轉成符號
    4. 補充

    在Ruby的世界裡面，字串（String）和符號（Symbol）都是內建的類別（Class）之一。那這兩個除了英文都是S開頭之外，還有什麼相同和不同之處嗎？

    相同點

    根據Ruby官方的API文件，可以看到String和Symbol有一些共同的方法

    capitalize, upcase, downcase

    >> :lailai.capitalize
 => :Lailai
>> :lailai.upcase
 => :LAILAI
>> :LAILAI.downcase
 => :lailai
>> "lailai".capitalize
 => "Lailai"
>> "lailai".upcase
 => "LAILAI"
>> "LAILAI".downcase
 => "lailai"

    empty?, start_with?, end_with?

    >> :lailai.empty?
 => false
>> :lailai.start_with?('a')
 => false
>> :lailai.start_with?('l')
 => true
>> :lailai.end_with?('i')
 => true
>> "lailai".empty?
 => false
>> "lailai".start_with?('a')
 => false
>> "lailai".start_with?('l')
 => true
>> "lailai".end_with?('i')
 => true

    length, size, []

    >> :lailai.length
 => 6
>> :lailai.size
 => 6
>> :lailai[0]
 => "l"
>> :lailai[1]
 => "a"
>> :lailai[2]
 => "i"
>> "lailai".length
 => 6
>> "lailai".size
 => 6
>> "lailai"[0]
 => "l"
>> "lailai"[1]
 => "a"
>> "lailai"[2]
 => "i"

    從irb（Interactive Ruby）的內容可以看到，上面這些方法不管是字串或是符號，都可以正常執行而不會噴錯，執行後的結果也都是相同的。

    不同點

    String可改變；Symbol不可變

    >> sym = :lailai
 => :lailai
>> str = "lailai"
 => "lailai"
>> sym[0]
 => "l"
>> str[0]
 => "l"
>> sym[0] = "a"
NoMethodError (undefined method `[]=' for :lailai:Symbol)
>> str[0] = "a"
 => "a"
>> sym
 => :lailai
>> str
 => "aailai"

    當我們嘗試改變:lailai的第一個字母時，發生了NoMethodError，這是因為Symbol沒有 [] = 這個方法。

    object_id

    在ruby的世界裡，每一個物件（object）都有object_id這個方法，會回傳該物件存在於記憶體位置裡的參照值﹐我們可以簡單把它當作該物件的門牌地址。

    讓我們來看看符號和字串的object_id的區別

    >> 10.times do
>>   puts :lailai.object_id
>> end
2044828
2044828
2044828
2044828
2044828
2044828
2044828
2044828
2044828
2044828
>> 10.times do
>>   puts "lailai".object_id
>> end
200
220
240
260
280
300
320
340
360
380

    在這邊可以看到，符號的object_id回傳的值每一次都是一樣的；而字串卻每次都不相同。

    如果想要讓字串的object_id固定的話，可以使用一個叫freeze的方法

    >> 3.times do
>>   puts "lailai".freeze.object_id
>> end
400
400
400

    這麼一來，這個”lailai”字串的object_id就會凍住了！

    相互轉換

    比較完字串和符號的異同之後，在這裡我們有個方法可以來讓它們彼此之間互相轉換

    to_s 轉成字串

    >> :lailai.to_s
 => "lailai"

    to_sym 轉成符號

    >> "lailai".to_sym
 => :lailai

    以上就是本篇的介紹，感謝各位的收看～

    補充

    另外，其實除了符號之外，每一個數字、布林值和nil值的object_id也都會是相同的。

    >> 3.times do
>>   puts 10.object_id
>> end
21
21
21
>> 3.times do
>>   puts true.object_id
>> end
20
20
20
>> 3.times do
>>   puts false.object_id
>> end
0
0
0
>> 3.times do
>>   puts nil.object_id
>> end
8
8
8
